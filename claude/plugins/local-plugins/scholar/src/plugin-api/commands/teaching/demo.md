# Create Demo Course

I'll help you create a complete demo course environment for testing and exploring scholar's teaching commands.

**Usage:** `/teaching:demo [path] [options]`

**Examples:**
- `/teaching:demo` - Create in ./demo-course/
- `/teaching:demo ~/my-test-course` - Create at custom path
- `/teaching:demo --verify` - Create and run verification tests
- `/teaching:demo --force` - Overwrite existing without prompting
- `/teaching:demo --quiet` - Minimal output (for CI)
- `/teaching:demo ~/test --force --verify` - Force + verify (CI-friendly)

**Options:**
- `--verify` - Run verification tests after creation
- `--force` - Overwrite existing directory without prompting
- `--quiet` - Minimal output (for CI)
- `--config PATH` - Reserved for future use (explicit config file path to use as template; currently ignored)
- `--debug` - Enable debug logging

## Implementation

Create a demo course directory with realistic STAT-101 content for testing all teaching commands.
Uses `scaffoldFlowDirectory` from `demo-scaffold.js` and `generateManifestFromConfig` from `manifest-generator.js` for `.flow/` setup.

### Step 1: Parse Arguments

Parse the command arguments:
- `path`: Target directory (default: `./demo-course/`)
- `--verify`: Run smoke tests after creation
- `--force`: Overwrite existing directory without prompting
- `--quiet`: Minimal output for CI/CD

### Step 2: Check Directory

If the target directory exists:
- If `--force` flag: Skip prompt, proceed to overwrite
- Otherwise: Ask user "Directory exists. Overwrite? (y/n)"
- If no, abort with message

### Step 3: Scaffold .flow/ Directory

Use `scaffoldFlowDirectory` to create the `.flow/` directory with config and manifest:

```javascript
import { scaffoldFlowDirectory } from '../../../teaching/commands/demo-scaffold.js';

const scaffoldResult = scaffoldFlowDirectory({
  targetPath: path,
  force: args.force,
  dryRun: args['dry-run']
});

// scaffoldResult contains:
// - success: boolean
// - filesCreated: string[] (paths to created files)
// - filesSkipped: string[] (existing files not overwritten)
// - weekStubsGenerated: number (draft stub weeks added)
// - warnings: string[]
```

This creates:
1. `[path]/.flow/teach-config.yml` — STAT-101 course configuration (from demo templates)
2. `[path]/.flow/lesson-plans.yml` — 15-week manifest (3 detailed weeks from template + draft stubs for weeks 4-15 generated by `generateManifestFromConfig`)

If `scaffoldResult.success` is false, show error and abort.

### Step 7: Write sample-student-work.md

Create sample submission for feedback testing:

```markdown
# Homework 3: Confidence Intervals
**Student:** Alex Johnson
**Date:** 2026-02-15

## Problem 1 (10 points)
A sample of 50 students had a mean GPA of 3.2 with a standard deviation of 0.5. Construct a 95% confidence interval for the population mean GPA.

**My Answer:**
The confidence interval is 3.2 ± 1.96(0.5/√50) = 3.2 ± 0.139 = (3.061, 3.339)

I used z = 1.96 for 95% confidence and the formula x̄ ± z(s/√n).

## Problem 2 (15 points)
Explain the interpretation of a 95% confidence interval. What does "95% confident" mean?

**My Answer:**
A 95% confidence interval means we are 95% sure the true population mean is in our interval. If we calculated the interval as (3.061, 3.339), there's a 95% chance the real mean GPA is somewhere in that range.

## Problem 3 (15 points)
If we wanted a narrower confidence interval, what are two ways to achieve this? Explain the trade-offs.

**My Answer:**
1. Use a larger sample size - this makes the standard error smaller
2. Use a lower confidence level like 90% - this makes the z-value smaller

The trade-off is that larger samples cost more money and time. Lower confidence means we're less sure about our interval.
```
